# ЛР-2

## Коллекция объектов

В проекте реализован контейнерный класс **CharacterCollection**, предназначенный для хранения и управления объектами класса **Character** из ЛР-1.

Коллекция позволяет работать с группой объектов: добавлять, удалять, искать, сортировать и фильтровать элементы.

Файл `demo.py` содержит несколько сценариев использования коллекции и демонстрацию всех реализованных методов.


## Реализация коллекции

### Контейнерный класс

Создан класс **CharacterCollection**, который:

* хранит объекты `Character`
* управляет коллекцией
* обеспечивает безопасный доступ к данным

Внутри используется список:

```python
self._items = []
```


### Добавление и удаление объектов

Реализованы методы:

* `add(item)` — добавление объекта
* `remove(item)` — удаление объекта
* `remove_at(index)` — удаление по индексу

Особенности:

* проверяется тип добавляемого объекта
* запрещено добавлять дубликаты (по имени)
* при удалении несуществующего элемента выбрасывается ошибка


### Доступ к данным

Методы:

* `get_all()` — возвращает список объектов
* `__getitem__()` — доступ по индексу

Пример:

```python
collection[0]
```


### Поиск объектов

Реализованы методы:

* `find_by_name(name)` — поиск по имени 
* `find_by_level(level)` — поиск по уровню 


### Итерация и длина коллекции

Реализованы магические методы:

* `__len__()` — позволяет использовать `len(collection)`
* `__iter__()` — позволяет перебирать коллекцию в цикле

Пример:

```python
for character in collection:
    print(character)
```


### Сортировка

Реализован универсальный метод:

* `sort(key)` — сортировка по заданному критерию

Пример:

```python
collection.sort(key=lambda x: x.level)
```

Сортировка выполняется по ключу, который определяет, по какому атрибуту сравниваются объекты.


### Фильтрация

Реализованы методы фильтрации, возвращающие новую коллекцию:

* `get_active()` — только доступные персонажи
* `more_health(min_health)` — персонажи со здоровьем выше заданного

Особенности:

* возвращается новая коллекция
* исходная коллекция не изменяется



### Инкапсуляция

Для защиты данных:

* список `_items` является приватным
* доступ к нему осуществляется только через методы
* метод `get_all()` возвращает копию списка, чтобы избежать изменения коллекции извне
